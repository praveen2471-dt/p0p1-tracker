Weekly analysis prompt — copy and paste into Claude
I've uploaded this week's P0/P1 incident data from DispatchTrack. Week label: [replace with e.g. "Week of Jun 23, 2026"] Please analyze all tickets and generate a full Excel report. For each ticket: 1. Extract the RCA from the comments — look for "RCA:", "Root Cause", "*RCA*" tags in the comment columns, or infer the root cause from how the engineer described the fix. Do not rely on the RCA column alone if one exists — the real detail is in the comments. 2. Categorize into exactly one of: Code bug / logic error | Infrastructure / devops | Configuration issue | Integration / API issue | Data / database issue | Performance / scalability | Security issue | Third-party / external | Feature gap | Unknown 3. Flag repeat issues: - "Yes" — same category AND same customer appeared in another ticket this week - "Possible" — same category pattern appears across multiple tickets - "No" — first occurrence of this category for this customer 4. Risk level — High / Medium / Low based on customer impact and recurrence potential. 5. Prevention action — one concrete, actionable sentence per ticket. Then generate a downloadable Excel file (.xlsx) with these 5 sheets: Sheet 1 — Issue Analysis Full ticket table: Issue Key, Customer, Priority, Week, Category, Sub-category, Repeated?, Risk, Status, Summary, RCA (extracted), Prevention Action. Color-code: category column by type, Repeated? column (red=Yes, yellow=Possible, green=No), Priority (red=P0, yellow=P1). Freeze row 1. Enable filters. Sheet 2 — Category Summary One row per category: Total tickets, Confirmed Repeat, Possible Repeat, First Occurrence, % of total. Sheet 3 — Repeat Patterns Group all repeated/possible tickets by category. For each group show: which customers are affected, all ticket keys, and a specific prevention recommendation for that category pattern. Sheet 4 — Customer Dashboard One row per customer: Total, P0, P1, Repeats, Repeat %, Top Category. Highlight customers with P0 incidents or repeat rate above 50%. Sheet 5 — Insights & Recommendations Key risks (repeat rate, import pipeline issues, config gaps) and 4-5 actionable product improvement recommendations based on this week's patterns. Professional formatting throughout: Arial font, color-coded badges, auto column widths, freeze panes.

Tips

Getting the best results

✓
Include all comment columns — when exporting from Jira/Google Sheets make sure Comment, Comment.1, Comment.2 … are all included. The RCA is in engineer comments, not usually the description.
✓
Set the week label in the prompt — replace [replace with e.g. "Week of Jun 23, 2026"] with the actual week before sending. It shows up throughout the report.
✓
Upload the file, don't paste — for sheets with more than 20 tickets, uploading the CSV as an attachment gives better results than pasting raw text.
✓
Save every week's Excel — download and archive each report. After a few weeks, upload two reports at once and ask Claude to compare them to track whether repeat rates and category patterns are improving.
